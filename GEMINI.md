# Gemini 작업 실행 가이드라인

이 파일은 LLM JIRA 시스템에서 작업을 실행할 때 따라야 할 규칙입니다.

## 작업 완료 기준

1. **완전한 구현**: 작업을 완료(DONE)로 표시하기 전에 다음을 확인하세요:
   - 모든 요구사항이 완전히 구현됨
   - 코드가 올바르게 동작함
   - 기존 기능이 손상되지 않음

2. **테스트 필수**: 모든 작업에는 테스트가 필요합니다:
   - 단위 테스트 작성
   - 엣지 케이스 고려
   - 에러 처리 확인
   - 모든 테스트 통과 확인

3. **상태 관리**:
   - 작업이 완전히 끝나고 테스트까지 통과했을 때만 DONE으로 표시
   - 해결할 수 없는 문제가 있으면 PENDING으로 표시
   - 작업 중인 상태는 IN_PROGRESS로 유지

4. **코드 품질**:
   - 깨끗하고 읽기 쉬운 코드 작성
   - 적절한 주석 추가
   - 일관된 코딩 스타일 유지
   - 보안 취약점 확인 (XSS, SQL Injection, Command Injection 등)

5. **문서화**:
   - 복잡한 로직은 설명 추가
   - API 변경사항 문서화
   - 필요시 README 업데이트

## 작업 실패 시

- 문제를 명확히 설명
- 시도한 해결 방법 기록
- 작업을 PENDING으로 표시
- 도움이 필요한 부분 명시

## 참고사항

- 항상 프로젝트의 전체 컨텍스트를 고려하세요
- 기존 코드 패턴을 따르세요
- 의문이 있으면 확인 후 진행하세요
